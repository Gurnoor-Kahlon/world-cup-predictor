# Data sources & how to connect real data

The project ships with **synthetic sample data** (`data/sample_matches.csv`,
generated by `data/generate_sample_data.py`) so it runs immediately. This guide
explains how to replace it with **real data** and which sources provide each of the
factors the model can use.

---

## The minimum: a match-results CSV

The predictor only *requires* a CSV of historical matches with these columns:

| column       | required | description                              |
|--------------|:--------:|------------------------------------------|
| `date`       | optional | match date (`YYYY-MM-DD`); enables form/rest |
| `home_team`  | ✅       | home team name                           |
| `away_team`  | ✅       | away team name                           |
| `home_score` | ✅       | goals scored by the home team            |
| `away_score` | ✅       | goals scored by the away team            |
| `tournament` | optional | competition name (affects Elo weighting) |
| `neutral`    | optional | `True`/`False` neutral venue             |

### The easy way — built-in scripts

```bash
# 1. download real, public data into data/raw/ (martj42 international results)
python scripts/download_data.py

# 2. validate + convert it to data/processed/matches.csv
python scripts/prepare_data.py
```

Once `data/processed/matches.csv` exists, the loader uses it **automatically**
(see `resolve_matches_path` in [`src/data_loader.py`](../src/data_loader.py)) — no
config change needed. Delete that file to go back to the bundled sample data.

`download_data.py` fetches **real** data (it does not fabricate anything); if you
have no internet it prints manual download instructions instead. You can point it
at any CSV in the same schema with `--url`.

### The manual way

Alternatively, overwrite `data/sample_matches.csv`, or pass an explicit path to
`load_matches(path=...)`. Drop large real datasets in `data/raw/` (git-ignored).

### Recommended free / legal sources for this CSV

- **Kaggle — “International football results from 1872 to 2017”**
  <https://www.kaggle.com/datasets/martj42/international-football-results-from-1872-to-2017>
  Already in *exactly* this schema (`date, home_team, away_team, home_score,
  away_score, tournament, city, country, neutral`). The easiest drop-in.
- **football-data.co.uk** — <https://www.football-data.co.uk/> Club league results
  **with historical bookmaker odds** (great for back-testing your probabilities).
- **github.com/openfootball** — open, community-maintained results in CSV/JSON.

## Sources for the *extra* factors

The model exposes clean hooks (extra CSV columns and the `context` argument to
`MatchPredictor.predict`) for richer factors. Here is where to get each one:

| Factor | Where to get it | Notes |
|---|---|---|
| **FIFA ranking** | <https://www.fifa.com/fifa-world-ranking/> | Monthly rankings; add as a `teams.csv` column. |
| **Elo ratings** | <https://www.eloratings.net/> (World Football Elo) | We compute our own, but you can seed from theirs. |
| **Recent form / results** | any results CSV above | Derived automatically from match history. |
| **Expected goals (xG)** | **FBref** <https://fbref.com/> · **StatsBomb open data** <https://github.com/statsbomb/open-data> | Best-quality public xG; feed via `home_xg_adjust`/`away_xg_adjust`. |
| **Injuries / suspensions** | **API-Football** <https://www.api-football.com/> · physioroom-style feeds | Pass counts via `context["home_injuries"]`. |
| **Player club form & minutes** | **FBref**, **Understat** <https://understat.com/> | Aggregate to a team strength adjustment. |
| **Squad market value** | **Transfermarkt** <https://www.transfermarkt.com/> | Respect their terms of use / robots.txt. |
| **Manager history** | Wikipedia / Transfermarkt | Encode as a feature if desired. |
| **Penalty-shootout record** | Wikipedia per-team shootout history | Pass via `context["home_shootout_winrate"]`. |
| **Weather / stadium** | **OpenWeather** <https://openweathermap.org/> ; venue databases | Map to an xG adjustment. |
| **Live fixtures & lineups** | **API-Football**, **Sportmonks** <https://www.sportmonks.com/> | Paid/free tiers; need an API key. |

---

## Suggested integration pattern

Keep raw pulls separate from the model:

1. Write a small `fetch_*.py` script per source into `data/raw/`.
2. Clean/merge into a single match CSV matching the schema above.
3. Point `config.MATCHES_PATH` at it (and add a `teams.csv` for metadata).
4. For per-match live factors (injuries, weather, rest), pass a `context` dict to
   `predict()` — no model changes needed.

```python
predictor = MatchPredictor().fit()
predictor.predict(
    "Brazil", "Germany", stage="Quarter-final", neutral=True,
    context={"home_injuries": 1, "rest_days_home": 3, "rest_days_away": 6,
             "home_shootout_winrate": 0.55},
)
```

> ⚠️ Always check each provider's **terms of service and licensing** before
> scraping or redistributing their data.
